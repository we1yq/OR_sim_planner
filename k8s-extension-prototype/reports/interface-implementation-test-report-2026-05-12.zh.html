MIGRANT 接口实现与单 A100 测试报告

日期：2026-05-12

集群：desktap RKE2 control-plane + rtx1-worker RKE2 GPU worker

Kubeconfig：~/.kube/or-sim-edge.yaml

测试 GPU：rtx1-worker 上的一张 NVIDIA A100-PCIE-40GB

总览

结果：单 A100 可验证的接口全部通过。

本轮实现内容：

  • 为 MigActionPlan.spec.podLifecyclePreview 增加真实 Pod lifecycle 执行入口。
  • 实现 createOrReuse：创建或复用请求 MIG 资源的 workload Pod。
  • 实现 reloadInPlace：通过 ConfigMap 投影卷更新 batch size，不删除 Pod。
  • 执行结果写回 MigActionPlan.status.validated.realPodLifecycle。
  • 补充 controller 对 Pod 和 Pod 日志的 RBAC。
  • 增加短时 CUDA profile workload runner，用于单 A100 profile-backed 测试。

本轮在 edge 集群中完成的测试：

  • A100 or-sim-empty -> all-3g.20gb -> or-sim-empty。
  • workload 调度到 nvidia.com/mig-3g.20gb。
  • batch size 从 4 热更新到 8，Pod UID 不变。
  • 使用 profile catalog 中的 resnet50 / profile=3g / batch=4 运行真实 CUDA 计算。
  • 清理临时资源，并确认 registry 回到 availableQueue。
重要边界：这次 resnet50 测试使用了 profile catalog 中的 workload/profile/batch，并在对应 MIG 分区中运行真实 CUDA 计算。 但它还不是完整的 ResNet50 模型服务，因为当前 repo 还没有真实模型服务镜像或模型仓库。

需要多张 A100 才能真实验证的部分

接口为什么需要多 A100
target_first reconfiguration需要旧 physical GPU 和新的 target physical GPU 同时存在。
多 GPU activeQueue 分配顺序单 A100 无法验证多 GPU 排序和竞争。
跨 GPU workload takeover需要旧实例和另一张 GPU 上的 takeover capacity。
多 GPU router cutover需要至少两个 serving target 才能验证流量切换。
takeover 后 drain 和 cleanup需要旧位置和新位置同时存在。
真实执行后的多 GPU canonicalization单 A100 无法验证 physical ID 复用、free-pool 策略和跨 GPU binding 保留。

需要外部 runtime 组件的部分

接口缺少的组件
真实 router/drain adapter还没有 router、queue 或 inflight request 来源；目前 drain 是 no-traffic stub。
精确 MIG UUID placementKubernetes device-plugin 默认按资源类型调度，不按指定 MIG UUID 调度；需要 scheduler/admission/CDI 或 assignment observer。
生产模型服务生命周期repo 里还没有模型服务镜像和 model repository。
镜像分发controller 仍使用 dev image；长期应使用内网 registry。
