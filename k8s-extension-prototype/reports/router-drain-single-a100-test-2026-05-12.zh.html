本次实现内容

  • 实现真实 Router/Drain 执行路径：controller/executors/router_drain_executor.py。
  • 增加 Router/Drain smoke ActionPlan 生成器：controller/test_harness/router_drain_smoke.py。
  • 增加轻量 HTTP router/workload 测试程序：tools/test_workloads/simple_router_workload.c。
  • 更新 adapter 结构文档，把真实运行组件和测试 harness 分开。

当前 adapter 可以停止源实例接收新请求、把新请求和排队请求切到目标 endpoint、等待源实例 inflight=0 且 queued=0，并把执行结果写入 MigActionPlan、WorkloadRoutePlan 和 ServingInstanceDrain。

本次测试证明了什么

  • 单张 A100 的两个 MIG 分区可以分别运行相同 workload 的两个 serving 实例。
  • 切换前流量进入源实例 A。
  • Router/Drain Adapter 可以让源实例停止接收新请求。
  • Router/Drain Adapter 可以把 router 目标改到目标实例 B。
  • Adapter 会等待源实例 inflight 和 queued 清零。
  • 切换后流量进入目标实例 B。
  • 执行结果会写回 Kubernetes CR。

仍需后续验证的内容

这次测试完成的是单 A100、分区到分区的真实 adapter 路径。下面内容需要多 GPU 或生产 serving runtime 接入后继续测试。

  • 跨物理 GPU routing。
  • 跨服务器 routing。
  • 生产模型 serving router 或负载均衡器集成。
  • 真实 queue/runtime 中排队请求的迁移语义。
  • 从生产 serving framework 获取 inflight metrics。
  • source、target 或 router endpoint 不可达时的故障处理。

当前 adapter 是 endpoint-based 设计，因此未来只要目标实例 endpoint 可达，同一套接口可以用于单卡分区、单机多 GPU 和多机 GPU 的 routing。
